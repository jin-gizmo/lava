
# Lava Commands and Utilities { x-nav="Commands & Utilities" }

The lava utilities are included in the
[jinlava](#python-executable-jobs) Python package together with the lava APIs.

Lava includes a number of CLI commands and utilities. Some of these are used by
the lava worker itself while others are support tools. Most of them can be used
stand-alone or invoked by lava [exe](#job-type-exe), [pkg](#job-type-pkg) and
[docker](#job-type-docker) jobs.

All of the utilities support a `-h` , `--help` option.

|Utility|Description|
|-|-|
|jinja|Render a Jinja template with specified parameters.|
|[lava-ami](#lava-ami-utility)|Manage the AMIs used in lava worker CloudFormation stacks.|
|[lava-backup](#lava-backup-utility)|Backup the [DynamoDB entries](#dynamodb-tables) for a specified lava realm.|
|[lava-check](#lava-check-utility)|Perform some basic health checks on DynamoDB table entries.|
|[lava-checksum](#lava-checksum-utility)|Set and validate checksums on lava DynamoDB table entries.|
|[lava-conn-usage](#lava-conn-usage-utility)|Generate an activity map of specified jobs over a defined time period.|
|[lava-dag-gen](#lava-dag-generator)|Generate a DAG specification for lava [dag](#job-type-dag) jobs from a dependency matrix.|
|lava-dispatcher|The lava job [dispatcher](#the-lava-dispatch-process).|
|[lava-dump](#lava-dump-utility)|Extract lava configurations from [DynamoDB](#dynamodb-tables) and dump them to files.|
|[lava-email](#lava-email-utility)|Send email using lava [email](#connector-type-email) connections.|
|lava-events|Query the lava [events](#the-events-table) table.|
|[lava-job-activity](#lava-job-activity-utility)|Query the lava realm [events table](#the-events-table) to map job activity over a specified time window.|
|[lava-new](#lava-new-utility)|Create a new [lava job framework](#the-lava-job-framework) project.|
|lava-ps|Show lava worker process information on the current host.|
|[lava-schema](#lava-schema-utility)|Perform deep schema validation for lava DynamoDB specification objects.|
|lava-sharepoint|Operate on SharePoint sites using lava [sharepoint](#connector-type-sharepoint) connections.|
|lava-slack|Send Slack messages using lava [slack](#connector-type-slack) connections.|
|lava-smb|Operate on SMB file shares using lava [smb](#connector-type-smb) connections.|
|[lava-sql](#lava-sql-utility)|Run SQL using lava [database connections](#database-connectors).|
|[lava-state](#lava-state-utility)|Manipulate lava state items.|
|[lava-stop](#lava-stop-utility)|Perform a controlled shutdown of the lava worker daemons. This also support AWS auto scaling lifecycle hooks.|
|[lava-version](#lava-version-utility)|Provides version information on the installed lava version.|
|lava-worker|The main lava worker.|
|[lava-ws](#lava-worker-status-utility)|Show lava worker status based on the worker SQS queues (queue depths, worker backlog etc.). This does not need to run on the worker host.|
|s3lambda|Trigger an AWS Lambda function by generating synthetic S3 event notifications.|
